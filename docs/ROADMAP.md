# AI Canvas 开发路线

本文档是项目唯一的长期路线入口。它只记录已经落地的能力、当前优先级和未完成方向；具体架构约束见 `DEVELOPMENT.md`，历史实施步骤不再长期保留。

## 当前基线

截至 2026-07-14，以下能力已落地：

- React Flow 多节点画布、历史撤销/重做、分组、布局、复制粘贴和连线派生数据。
- 图片/视频生成、LLM 输出节点和可恢复任务队列。
- 暗色、浅色、跟随系统主题。
- 项目快照 schema 版本和集中迁移入口。
- 项目级工作区索引、单项目加载、保存和删除；启动及项目切换按需加载。
- 工作区目录包导入导出，覆盖项目、任务队列、脱敏配置和被引用资产。
- 单项目目录包导入导出，支持 ID 冲突预检、导入副本和显式替换。
- Electron 桌面壳、安全 preload 隔离、Web/Desktop `platformBridge` 选择入口，以及首版原生工作区 IPC/Node 文件服务。
- 大图缩略图 LOD、手动性能模式、交互期本地拖拽状态和浏览器性能采样。
- 轻量单测、浏览器冒烟、Lint、类型检查和生产构建验证链路。
- Toolbar、项目管理、GenerateNode、图片编辑器、核心 store 和 Web 持久化模块的第一轮职责拆分。

## 当前阶段

P0 Electron 桌面文件边界、P1 质量与数据治理、P2 桌面 SQLite 存储、P3 扩展能力和 P4 稳定性与发布收尾已完成。本仓库进入本地 Web/Electron 维护阶段，继续保持 `platformBridge`、项目快照和跨平台目录包契约稳定。

后续改动优先修复本地工作区可靠性、桌面发布、兼容迁移和真实使用反馈。组件、节点、Zustand store 和 feature orchestrator 继续只依赖 `platformBridge`。

## P0：桌面文件边界（已完成）

目标：桌面端不再依赖浏览器 File System Access API，同时继续使用可迁移的项目 JSON 和文件资产模型。

验收标准：

- 原生目录选择、项目读写、配置读写、资产读写和目录包导入导出全部通过 `platformBridge`。
- Web 端行为不变，业务 store 不出现 Electron API、Node 文件 API、绝对路径或 SQLite 依赖。
- 桌面窗口可完成工作区选择、项目创建、保存、关闭重启恢复和目录包往返。
- `npm run test`、`npm run lint`、`npm run build`、`npm run test:smoke` 和桌面手测通过。

闭环验证由 `npm run desktop:smoke` 和 `npm run desktop:smoke:ui` 固化；后者使用隔离工作区在真实 Electron 窗口执行目录包往返与关闭重启恢复，并验证导出配置不含 Provider API Key。

## P1：质量与数据治理（已完成）

- 已完成：增加 Web/Desktop 真实磁盘占用扫描、未引用资产路径预览、缺失引用统计和清理前影响说明。
- 已完成：项目归档/恢复，覆盖持久化归档状态、活动项目安全回退、独立归档视图和恢复入口。
- 已完成：按项目导入、ID 冲突检测和显式冲突处理；副本模式生成新 ID 并隔离资产路径，替换模式保留原 ID。
- 已完成：完善 Provider 契约测试、快照迁移 fixture 和工作区兼容 fixture；固定阿里/OpenAI 兼容图片与聊天请求响应契约、无版本快照迁移，以及单体/分项目工作区读取行为。
- 已完成：建立统一错误与诊断入口，覆盖保存、模型调用、网络和资源恢复失败；提供分类、错误码、可重试标记、脱敏上下文、会话去重与诊断面板。
- 已完成：补工具栏、弹层、菜单、节点按钮的焦点顺序、ARIA 标签和键盘流；统一模态焦点圈定/返回、菜单方向键、Escape 关闭和工具组语义。
- 已完成：为历史栈总量和大文本增加显式治理流程；统计撤销/重做快照字节数和大文本来源，提供确认后清空历史，并移除历史条数与文本附件的静默截断。

## P2：桌面原生存储与安全（已完成）

桌面文件边界稳定后按以下顺序实施：

- 已完成：SQLite 保存项目索引、快照元数据、任务、资产索引、设置和审计日志；数据库使用 schema 版本并在升级和整包替换前备份。
- 已完成：项目快照第一阶段仍整体保存 JSON，不拆节点和边表。
- 已完成：图片、视频、缩略图和预览图继续使用文件目录，不写入 SQLite blob。
- 已完成：提供 Web JSON 工作区首次打开导入桌面数据库，以及桌面数据库导出兼容目录包的双向迁移。
- 已完成：桌面服务商设置和 Provider API Key 统一保存在 SQLite；浏览器缓存、诊断报告和跨平台导出包继续强制脱敏，不引入系统密钥链。

任何新存储格式都必须具备版本、迁移、备份和跨平台导出路径。

## P3：扩展能力

- 已完成：节点注册协议统一节点类型、稳定 ID 前缀、兼容渲染别名、默认创建工厂、连接规则、输出布局和工具栏/右键入口。
- 已完成：工作流模板支持保存和复用局部节点组合，重建稳定 ID、内部连线和节点引用，并通过 Web 工作区文件、桌面 SQLite 与跨平台目录包持久化。
- 暂缓：执行编排（按依赖运行、批量重跑、跳过、重试和中断恢复）当前投入与实际收益不匹配，待真实工作流需求明确后再评估。
- 已完成：本地审计和诊断查询支持桌面 SQLite 操作日志、分类/时间/关键词筛选、分页和会话诊断双视图，并保持敏感信息脱敏边界。
- 已完成：跨项目全文搜索和资产索引支持项目、文本、提示词、LLM 输出及资产元数据检索，桌面 SQLite schema v2 持久化索引，Web 端使用同一提取契约兼容查询，结果可打开项目并定位节点。

远端同步或团队协作需要独立的服务端数据与协作协议，不由本地 SQLite 承担。

## P4：稳定性与发布收尾

- 已完成：浏览器 smoke 的项目菜单动作改为在页面线程中重新查询当前 DOM 后连续触发，项目打开后明确等待管理弹层关闭，避免自动保存刷新项目卡片造成元素脱离。
- 已完成：Electron UI smoke 统一禁用截图动画、使用 60 秒超时和一次短延迟重试，并复用不持有旧项目卡片元素的菜单动作；源码 Electron 工作区完整往返已通过。
- 已完成：`npm run desktop:build` 已通过受信任的 HTTP/Mixed 代理建立 HTTPS 下载链路并生成 Windows 便携版；解包产物 UI smoke 已通过。此前的 `127.0.0.1:443` 连接拒绝来自错误的下载链路，不是应用构建失败或必须绕过的 TLS 证书问题。
- 已完成：增加 Windows NSIS 辅助安装包，支持安装范围与目录选择、桌面/开始菜单快捷方式和标准卸载；隔离安装、已安装程序 IPC smoke 与卸载闭环已通过。代码签名和正式发布渠道仍不在当前范围。

## 网站端边界

账号网站、PostgreSQL 关系化项目图、对象存储和服务端任务已迁移到独立仓库维护：

- 仓库：<https://github.com/136909482/ai-canvas-cloud>
- 网站端路线：<https://github.com/136909482/ai-canvas-cloud/blob/main/docs/ROADMAP.md>
- 数据模型：<https://github.com/136909482/ai-canvas-cloud/blob/main/docs/DATA_MODEL.md>

本仓库不新增 Cloud API、PostgreSQL、Redis、Worker、账号系统或 OSS 实现。需要跨仓库协调的长期契约仅包括：

- 版本化 `ProjectRecord` 与 `{ canvas, taskQueue }` 快照语义。
- 工作区和单项目目录包格式。
- Provider API Key 导出脱敏。
- 本地资产相对路径及目录包中的引用完整性。
- 旧快照/旧工作区 fixture 的追加兼容。

网站端通过显式导入读取本仓库导出的目录包；登录、退出或网络恢复不得让本仓库自动上传本地工作区。网站端的内部表结构、API、任务和部署计划只在 `ai-canvas-cloud` 维护。

## 暂不进入当前范围

- 自动更新、代码签名和正式安装包渠道。
- 项目合并导入和 ZIP 单文件封装。
- 账号、团队、实时协作、跨组织分享、商业计费和云端同步；这些能力属于 `ai-canvas-cloud`。
- 自动按节点数量切换性能模式。
- 把节点或媒体内容隐藏、留白来换取性能。

## 路线维护规则

- 完成一个阶段后直接更新本文件的状态，不新增长期保留的“完成计划”文档。
- 复杂功能在实施期间可以使用临时计划，但合并前必须把有效结论写回长期文档并删除临时计划。
- 路线优先级变化时，同时检查 `README.md`、`DEVELOPMENT.md` 和 `DESKTOP.md` 是否仍然一致。
